Glossary

HTML
As we all know that HTML is one of the fundementals methods on programming and designing websites. For those who don't know HTML stands for Hypertext Markup Language. Below i will be describing all differne types of necessary tags, when it comes to web development.
CSS
CSS stands for Cascading Style Sheets. CSS describes how HTML elements are to be displayed on screen, in other word HTML is known as a skeleton whereas CSS is the organs and the skin. Liz has showed us how important mastering the understading of CSS really is.
Root Directory
When it comes to accessing a certain file in web development, maintaining a specific structured root directory. For example a link to a file called home.html within a root directory can be accessed using "/home.html"./
Commenting
During our web development classes emphasised on the importance of commenting, stating how developers utilise this method to keep track of their complex code. When commenting in HTML by inputting (!-- hi this is the commenting space --) i.e replacing the curly brackets with the alligator brackets.
Syntax
The definition of syntax can be a generic term used, but in this context it is the set of rules and tags required that defines the specific combination elements required to portray a digital content i.e website.
Tags
HTML & CSS tags are simply key elements required to correctly structure the site provide the perfect correlation between the code that you have written and what is presented to users.
Page Structure
When it comes to desiging websites, it requires a variety of key elements that provide certain professionalism to the website, making sure some users are engaged to the site. For example, images, text, video, titles are just some of the key elements required.
Responsive Web Design
Responsive web design is an approach to web designs that makes web pages professional well on a variety of devices or screen sizes.
Attribute
The class attribute specifies one or more classnames for an element. The class attribute is mostly used to point to a class in a style sheet.
Inserting Links
Simply a link, is a unique reference that the user can specifically follow either by clicking. By embedding a link in a text, image, or even video it allows users to fully navigate your website.
Web standard bodies
In the web deveopment section ('W3 & WHATWG') are two of the most common standardization bodies, and their is to set the rules for the web.
CSS - Flexbox
Flexbox is a layout mode intended to accommodate different screen sizes and different display devices. Using the property allows you to simply to lay out and align certain items on specific locations on a page.
Padding
The padding is the spacing between the content and the border edge of the element. We can adjust this value with CSS to move the border closer to or farther from the content.
Margin
The margin is the space around the element. The larger the margin, the more space between our element and the elements around it. We can adjust the margin to move our HTML elements closer to or farther from each other.
DIV Tag
A block level container (or 'division' of the web page) for content with no semantic meaning. However it allows simplicity for developers when handling complex codes.
Paragraphs
One of the most common tags in HTML - it denotes a paragraph of text. It often has other elements nested inside of it such as imags links etc.
Tables
An element for displaying information in rows and columns. Supports headers and footers for labeling columns. Divides information into rows (denoted by the tr tag) which contain cells (denoted by the td tag).
Title Tag
This tag displays the page title at the top and tells search engines what the title of your site is. It goes inside (head) tags.
Forms
A webform, web form or HTML form on a web page allows a user to enter data that is sent to a server for processing.
Media Queries
Media queries are a feature of CSS that enable webpage content to adapt to different screen sizes and resolutions. ... Therefore, the default styles are typically defined first within a CSS document, followed by the media queries for different screen sizes.
Hover
The :hover tag matches when the user interacts with an element with a pointing device, but does not necessarily activate it. It is generally triggered when the user hovers over an element with the cursor to implement certain additions.
Typography
Web typography refers to the use of fonts on the World Wide Web. When HTML was first created, font faces and styles were controlled exclusively by the settings of each Web browser. During our web clss Liz showed us the standards for web typography.
Colours
For colour usage in HTML and CSS there are differnt forms to implememnt colours on a site. For example Hexacdecimal colours that contain '#' followed by a 6 digit code.
Images
During our classes once again Liz mentioned that when desiging websites copyrighted images should NOT be used for you sites due to legal issues. There are also different form of images like pixalated or vector images. Also is key to sustain the correct image format.
Microinteraction
Microinteraction were taught to us in class. Liz mentioned the 4 pillars trigger,feedback and loops like for example switches that can be used on mobile sites to make it more advanced for users.
Video
When it comes to creating a responsive site adding a video is one of the key elements due to the fact it would enhance user-friendliness providing user with new elements such as a video.
NAV
Represents navigation for a document. The nav element is a section containing links to other documents or to parts within the current document. Not all groups of links on a page need to be in a nav element.
Audio
The (audio) tag defines sound, such as music or other audio streams. Which once again provides a great addition for different forms of elements.
Footer
The footer is another key design of a site. For example there are certain elements like forms that are added to the footer section. The footer instinctly lets users know that the footer section is the end of a site without stating as a footer.
Meta
The (meta) tag provides meta-data about the HTML document. Metadata will not be displayed on the page, but can be read by certain bots on search engines to ensure its suitability.
DOCTYPE
The doctype declaration specifies which version of HTML is used in a document. It has a direct effect on whether your HTML will validate.
Element
A HTML element is everything from the start tag to the end tag. Using the (p) tag as an example: (p) tag This is a paragraph(/p) the (p) element has a start tag (p) and an end tag (/p) with the element content being ‘This is a paragraph’. The (p) element defines a paragraph in the HTML document.
Navigation
Navigation refers to the system that allows visitors to a website to move around that site. Navigation is most often thought of in terms of menus, but links within pages, breadcrumbs, related links, pagination, and any other links that allow a visitor to move from one page to another are included in navigation.
OPTIMISE
Has two possible meanings in web design. The first is website/page optimisation. This relates to how the page is structured (both code and content) with regard to search engines. A well optimised website is search engine friendly. The second meaning relates to graphics and pictures that are used on websites.
CSS - Transition
transitions allows you to change property values smoothly (from one value to another), over a given duration.
Browser
browser is the piece of software on your computer/phone/Internet enabled device that lets you view web pages. It's the window you open when you want to go on the Internet and view a web page. Popular mainstream browsers include Internet Explorer, Firefox, Safari and Chrome.
User Interface (UI)
The user interface is simply how a user interacts with the design on a page. For example, the battery indicator on your smart phone is a part of the user interface.
Usability
If a website is hard to use, people will abandon it. You want your site to retain users, of course, so the term "usability" refers to how easy it is to use and navigate for an average person.
Front-End
The front-end of a website refers to part of the site that your user interacts with directly. Coding languages like HTML and CSS are parts of the site that encompass the front-end.
Back-End
The back-end of a website is everything that the user can't see and interact with. Typically back-end coding languages like PHP or .NET are run on a server. For this reason, the back-end code is also sometimes referred to as server-side code.
Domain
This is the name of your website, or what people type into the address bar when they want to find your site. For example my site could be called www.modevelopment.com
eCommerce
An eCommerce website is site where things are purchased, like an online store. Amazon.com is a massive eCommerce site.
PHP
PHP is the coding language mainly used in WordPress. It’s a dynamic language that gives lots of flexibility for different types of functions.
favicon
An icon associated with a particular website, typically displayed in the address bar of a browser accessing the site or next to the site name in a user's list of bookmarks.
Junior Web Designer
A web designer/developer is responsible for the design, layout and coding of a website. They are involved with the technical and graphical aspects of a website - how the site works and how it looks.
Mobile Application Developer
Mobile developers are a type of software developer. They specialise in mobile technology such as building apps for Google's Android, Apple's iOS and Microsoft's Windows Phone platforms.
Software Engineer
Becoming a software developer, also known as a computer programmer, you'll be playing a key role in the design, installation, testing and maintenance of software systems.
Video Game Designer
Game designers have duties like designing characters, levels, puzzles, art and animation. They may also write code, using various computer programming languages.
IT Security Specialist
When it comes to IT security specialist their main role is to analyse where security breaches may occur or have occurred, and repair or strengthen systems against such breaches. This relates to the systems and networks used by companies and organisations to manage their information and information technology.
Web Developer
A developers role is responsible for designing, coding and modifying websites, from layout to function and according to a client's specifications. Strive to create visually appealing sites that feature user-friendly design and clear navigation
Networking Administrator
Network and computer systems administrators are responsible for the day-to-day operation of these networks. They organize, install, and support an organization's computer systems.
